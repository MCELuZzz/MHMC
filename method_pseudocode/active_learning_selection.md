# Pseudocode: diversity-driven active-learning selection

The public package includes a reference implementation of the SOAP/KPCA/FPS selection module in `scripts/selection_reference/`.

## Workflow

```text
1. Read the candidate pool generated by the sampler.
2. Assign or read source labels for each candidate structure.
3. Compute an averaged SOAP descriptor for each structure.
4. Reduce the descriptor matrix using kernel PCA.
5. Split the pool into source strata according to source labels.
6. Allocate a fixed quota to each source stratum.
7. Within each stratum, perform farthest point sampling in KPCA space.
8. Merge all selected candidates into one DFT-labeling batch.
9. After DFT labeling, merge converged labeled structures into the training set.
10. Retrain the MLIP and repeat for the next active-learning round.
```

## Rationale

The selection module is intentionally decoupled from the MLIP architecture. It does not require committee models and can be used with NequIP, MACE, SchNet, or other MLIP backbones, provided the candidate structures can be read by ASE.
